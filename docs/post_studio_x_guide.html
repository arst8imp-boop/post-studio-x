X専用・AI下書きツール

Post Studio X
使い方ガイド

X（旧Twitter）の投稿を、アカウントごとの語り口で下書きするローカルツール。
長文・短文・引用・リプ・会話リプ・X記事を、過去投稿を参考にしながら並べて生成できます。

ローカルで動く サブスク認証優先＝追加課金なし ポート 7879
Post Studio X1 / 11
OVERVIEW

これは何？ ／ できること

あなたの代わりに「そのアカウントらしい」Xの投稿を下書きするツールです。

アカウントごとの語り口

タロ／しーぷ／馬井など、キャラ設定に沿った文体で生成。新しいアカウントも自分で作れます。

6つの投稿タイプ

長文・短文・引用ポスト・リプライ・会話リプ・X記事。用途に合わせて選ぶだけ。

過去投稿を参考（RAG）

自分の過去ポストを参照して、語り口や数字のブレを抑えます。

並べて一括生成

ペインを複数開いて「まとめて生成」。バズ拾い・ネタ出し・履歴・カバー画像も。

X（旧Twitter）専用に特化した下書きツール。余計な設定は省き、Xの投稿づくりだけに集中できます。
Post Studio X2 / 11
REQUIREMENTS

用意するもの（自分でそろえる）

DB の URL や API キーは 各自で取得して .env に入れます（共有はしません）。

1
Claude Code … 生成の認証に使用。claude.com/claude-code からインストールしてログイン。
2
Python 3.12 以上 … 3.9 では動きません。brew install python@3.12
3
Neon（無料のクラウドDB）の接続文字列 … neon.tech で無料登録し Connection string をコピー。
4
（任意）Anthropic / Gemini API キー … サブスク上限時の予備・カバー画像生成用。
生成は Claude Code のサブスク認証を優先するので、通常は API 課金なし（¥0）で使えます。
Post Studio X3 / 11
SETUP

セットアップ

GitHub からコードを取得して、5 手順で起動します。

# 1) コードを入手（Gitが無ければ GitHub の Code → Download ZIP でも可） git clone https://github.com/arst8imp-boop/post-studio-x.git cd post-studio-x # 2) 仮想環境と依存 python3.12 -m venv .venv .venv/bin/pip install -r server/requirements.txt # 3) .env を作って自分の値を入れる（Neon の URL / 任意でAPIキー） cp .env.example .env # 4) DB作成 → 5) 起動（http://localhost:7879 が開く） .venv/bin/python3 setup_db.py ./start.sh
Claude Code に任せてもOK：clone したフォルダを開いて「このツールをセットアップして起動して。Neon の URL と API キーは用意してある」と頼めば、2〜5 を代わりにやってくれます。
Post Studio X4 / 11
SETUP · .env

.env の中身

post_studio_x/.env に置きます（無ければ親フォルダの .env でも可）。

DATABASE_URL=postgres://...neon.tech/neondb?sslmode=require API_MONTHLY_BUDGET_JPY=0 # 0 = API課金を既定でブロック（安全） # ここから下は任意 ANTHROPIC_API_KEY=sk-ant-... # サブスク上限時の予備 GEMINI_API_KEY=AIza... # カバー画像を使うときだけ
DATABASE_URL は必須（履歴・アカウント・使用量の保存先）。キー類は無くてもサブスク認証で動きます。
Post Studio X5 / 11
SCREEN

画面の見方

画面は「ペイン」の集まり。ヘッダーの各ボタンから機能を開きます。

ペイン（生成の単位）

上で アカウント と 投稿タイプ を選び、テーマ欄に指示を書いて 生成。

ヘッダーのボタン

コピー／カバー

生成後に コピー でクリップボードへ。X記事タイプなら カバー 画像も作れます。

Post Studio X6 / 11
GENERATE

生成の基本

6つの投稿タイプから選んで、テーマ欄に指示を入れるだけ。

長文ポスト短文ポスト（140字）引用ポスト リプライ会話リプ（流れ継続）X記事（長文）
A
引用・リプ・会話リプ … テーマ欄に「相手の元ポスト」をそのまま貼り付ける。
B
まとめて生成 … 複数ペインにテーマを入れて一括生成。案出しに便利。
C
追加指示欄 … 「もっと短く」「数字を入れて」など細かい注文を任意で。
生成物は自動で 履歴 に保存されます。あとから履歴 → クリックでペインに復元できます。
Post Studio X7 / 11
FEATURES

便利機能

バズ

伸びているポストを一覧。引用RT用／リプ用にワンクリックでペインへ流し込み。

ネタ出し

過去の生成履歴を踏まえ、まだ書いていない投稿ネタを提案。気に入ったら「ペインに流す」。

履歴

生成物の一覧・復元。ブランドで絞り込みも可能。

カバー画像

X記事タイプで本文から3行を抽出→カバー画像を自動生成（Gemini キーが必要）。

Post Studio X8 / 11
ACCOUNTS

アカウント管理

ヘッダーの「アカウント」から、Xアカウント（ブランド）を管理します。

1
新規作成 … 表示名を入れて「作成」。ペインのプルダウンから選べるようになります。
2
名前変更 … 作成したアカウントの表示名を変更。
3
削除 … 確認のあと削除。「履歴も一緒に消すか」をその場で選べます。
組み込みの3アカウント（タロ／しーぷ／馬井）は削除・名前変更できません。
Post Studio X9 / 11
COST & SAFETY

コストと安全（重要）

「勝手に課金されない」よう、既定でAPI課金をブロックしています。

1
サブスク認証を優先 … 通常は Claude Code のプラン枠で生成＝追加課金なし（¥0）。
2
予算0円でブロック … サブスク上限時も、API_MONTHLY_BUDGET_JPY=0 の間は課金せず生成を停止。
3
必要なときだけ解除 … 右上「今月のAPI課金」→ ブロック解除トグルでAPI課金を手動許可（確認あり・既定はOFF）。
使用量モーダルでは、今日／今月／累計のトークン数と課金額、予算バーが確認できます。
Post Studio X10 / 11
TROUBLESHOOTING

困ったら

?
ポートが使用中 … 別ポートで起動：POST_STUDIO_PORT=7880 ./start.sh
?
TypeError 等で起動しない … Python が 3.12 か確認。3.9 なら venv を作り直す。
?
履歴・アカウントが保存されない … .env の DATABASE_URL（Neon）を確認。
?
カバー画像で 429 … Gemini 無料枠切れ。時間を置くか別モデルを指定。
迷ったら、このガイドごと Claude Code に渡して「この通りにセットアップして」と頼めば大丈夫です。
